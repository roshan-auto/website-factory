# Refine UI Workflow

## Goal
Improve the visual quality of an existing page or website until it meets a professional standard.

## Steps

1. Open the target page
- Use local dev server or preview URL
- Focus on homepage first unless specified

2. Visual inspection
Check for:
- spacing inconsistencies
- weak visual hierarchy
- poor typography scaling
- inconsistent button styles
- alignment issues
- sections feeling “generic” or templated
- mobile responsiveness issues

3. Identify top issues
- List the top 5 most noticeable UI problems
- Prioritize high-impact fixes

4. Apply improvements
- Fix spacing rhythm (padding, margins, section gaps)
- Improve typography (sizes, weights, contrast)
- Improve layout structure (grid, alignment)
- Enhance CTA visibility
- Remove unnecessary clutter

5. Responsive pass
- Verify mobile layout
- Fix overflow, stacking, spacing issues

6. Repeat (max 3 passes)
- Re-check after changes
- Improve again if needed
- Stop after 3 iterations or when only minor subjective issues remain

7. Output
- List changes made
- List remaining minor improvements (if any)
- Do not loop indefinitely

## Rules
- Do not rebuild from scratch unless necessary
- Focus on polish, not feature expansion
- Avoid overdesign
- Keep UI clean and conversion-focused
