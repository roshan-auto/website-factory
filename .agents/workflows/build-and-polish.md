# Build and Polish Workflow

## Goal
Create a new website and refine it to a high-quality, production-ready level using structured steps.

## Steps

1. Build phase
- Run the "new-site" workflow
- Ensure a working site is created

2. First polish pass
- Run "refine-ui" on the homepage
- Improve layout, spacing, typography, and hierarchy

3. QA pass
- Run "browser-qa"
- Collect issues and observations

4. Second polish pass
- Fix the top issues found in QA
- Improve responsiveness and visual consistency

5. Final check
- Ensure:
  - no obvious layout issues
  - mobile responsiveness is good
  - CTA is clear
  - page looks clean and professional

6. Output
- Summary of what was built
- Key improvements made
- Remaining minor improvements (if any)

## Rules
- Do not skip steps
- Do not loop indefinitely
- Focus on high-impact improvements
- Stop when quality is clearly above average, not perfection
